# Open Questions

Decisions **blocked on information someone has to go get** — from a teammate, a vendor, a customer, or
a system we don't yet have access to. Not a general TODO list (that's the backlog) and not a debt log
(that's [TECH-DEBT.md](TECH-DEBT.md)). An item lives here only because Claude *can't* resolve it from
the repo — it needs an answer from outside.

> When a session touches a related area — or the user mentions an upcoming meeting — surface the
> relevant items proactively. When an answer arrives, move the item to **Resolved** with the answer +
> date.
