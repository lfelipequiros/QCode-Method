# Tech Debt

Deliberate shortcuts we chose to take — each recorded with the clean target it deviates from and the
**paydown trigger** (the concrete condition that means "now fix it"). A shortcut you can name and
track is a decision; one you hide is rot. Untriggered debt is how "temporary" becomes forever.

> Logged by `tech-planning` (planned shortcuts) and `tech-build` (forced shortcuts found mid-build),
> and reconciled by `tech-qa` (an unlogged shortcut in a diff is a review failure). Each entry links
> from its backlog story.
